# Frontend Design System

This document defines the role of the future frontend design system and theme documentation for `apps/web`.

## Purpose

The frontend design system should provide a shared visual and interaction language for:

- marketing pages
- authenticated product pages
- reusable UI components
- layout and spacing decisions
- typography and color rules

## Documentation Scope

This folder should eventually capture:

- theme principles
- color roles and token definitions
- typography hierarchy
- spacing rhythm
- shell and layout patterns
- component usage rules
- motion and interaction standards

## Initial Design Priorities

When implementation starts, the frontend should document:

- how brand expression differs between marketing and dashboard surfaces
- which design tokens are shared across both surfaces
- where distinct patterns are allowed
- how future themes, mockups, and design explorations are recorded

## Relationship To Existing Docs

The custom docs UI in `docs/v2` is an existing documentation interface.

It is not the future product design system.

The frontend design system documented here should focus on the future web product experience, not the current docs renderer.

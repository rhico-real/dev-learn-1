## What I Did

### Task 1 ‚Äî Architecture & Setup (Mar 18)
Started with architecture first, code second. Went with a bounded-context / domain-driven folder structure (`src/domain/identity/`, `src/domain/organization/`, etc.) ‚Äî the kind of structure that holds up as the app grows instead of becoming a mess at scale.

Designed the Prisma schema upfront: `User`, `Organization`, `OrgMembership`, `Event`, `EventRegistration`. Set up Docker Compose with PostgreSQL and Redis.

### Task 2 ‚Äî Prisma + Database (Mar 20)
Defined the full schema, ran first migrations, wired up `PrismaService` and `RedisService` into the NestJS app.

### Task 3 ‚Äî Auth Module (Mar 20‚Äì21)
Built the full auth flow ‚Äî register, login, refresh token, logout. The important part: logout actually *works*. JWTs are stateless so you can't cancel them server-side by default. The fix is storing refresh tokens in Redis and deleting them on logout, so a revoked token can never be reused.

Set up `JwtStrategy` with Passport for protected route validation.

### Task 4 ‚Äî User Module (Mar 21)
`GET /users/me`, `PATCH /users/me`, `DELETE /users/me`. Used a `@CurrentUser()` decorator to pull the authenticated user from the JWT payload ‚Äî cleaner than passing IDs manually through every controller method.

### Task 5 ‚Äî Shared Infrastructure + Tests (Mar 22‚Äì23)
Built `SharedModule` ‚Äî `JwtAuthGuard`, `RolesGuard`, `TransformInterceptor`, `LoggingInterceptor`, `HttpExceptionFilter`. Then wrote unit tests for services and guards, plus E2E tests covering the full auth flow.

Hit a real bug in the E2E tests: the logout test was passing when it shouldn't. Turned out the Redis write (storing the invalidated token) wasn't being awaited before the next request fired. The test was essentially racing itself. Fixed it ‚Äî the test now actually proves what it's supposed to prove.

## Struggles

**TypeScript's decorator patterns in NestJS** ‚Äî coming from Dart, the syntax is close but NestJS leans heavily on decorators in ways that aren't immediately intuitive. Took a few repetitions before it stopped feeling like magic.

**NestJS DI vs just importing modules** ‚Äî the module/provider declaration pattern is more verbose than I'm used to. The benefit is clear (explicit dependencies, testability) but the mental overhead is real until it clicks.

**The async race condition** in the E2E test was the most interesting one. The test was structured correctly but the timing was wrong. A missing `await` on the Redis call was the entire problem.

Subject: Week 1 ‚Äî Architecture decisions, real auth, and a sneaky test bug

Building RunHop in public. Week 1 is done.

The stack is NestJS, PostgreSQL, Redis, and Prisma. The goal is to build it properly ‚Äî not to get something working fast and fix the mess later.

Week 1 was mostly foundation work: architecture design, database setup, full auth system, and tests. Not glamorous but it's the stuff that determines whether the codebase is still maintainable in three months.

The most interesting thing this week was the auth system. JWT-based auth is everywhere, but most implementations skip a real logout mechanism. Stateless tokens can't be invalidated by default ‚Äî once issued, they're valid until they expire. The solution is Redis: store refresh tokens server-side, delete them on logout. Simple in concept, easy to get wrong in execution.

I got it wrong once. My E2E test for logout was passing when it shouldn't have been. Took me longer than I'd like to admit to find the problem: a Redis write wasn't being awaited, so the invalidation hadn't happened yet when the next request fired. Classic async timing bug, hidden by the fact that the test *usually* passes fast enough that it doesn't matter.

Fixed it. The test is honest now.
